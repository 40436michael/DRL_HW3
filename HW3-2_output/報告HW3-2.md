# 4. Double DQN

## 4.1 Double DQN 概念

傳統 DQN：

```text
使用同一個 network：
1. 選 action
2. 評估 Q-value
```

容易造成：

```text
Q-value overestimation
```

Double DQN 則分成：

| Network | 功能 |
|---|---|
| Main Network | 選 action |
| Target Network | 評估 action |

---

## 4.2 Double DQN 架構

```python
class DoubleDQN(nn.Module):
```

網路架構：

```python
64 → 128 → 64 → 4
```

Activation：

```python
ReLU()
```

---

## 4.3 Double DQN Target 計算

先由 Main Network 選 action：

```python
next_actions = torch.argmax(
    model(next_states),
    dim=1
)
```

再由 Target Network 評估：

```python
next_q_target = target_model(next_states)

next_q = next_q_target.gather(
    1,
    next_actions.unsqueeze(1)
).squeeze()
```

最後：

```python
target_q = rewards + gamma * (
    1 - dones
) * next_q
```

---

## 4.4 Double DQN 優點

相比 Naive DQN：

- 減少 Q-value 高估
- 提升訓練穩定性
- 改善收斂品質

# 5. Dueling DQN

## 5.1 Dueling DQN 概念

Dueling DQN 將 Q-value 分解成：

```text
Q(s,a) = V(s) + A(s,a)
```

其中：

| 名稱 | 說明 |
|---|---|
| V(s) | State Value |
| A(s,a) | Advantage Function |

---

## 5.2 Dueling DQN 架構

本次使用：

```python
class DuelingDQN(nn.Module):
```

分成：

### (1) Feature Layer

```python
self.feature
```

### (2) Value Stream

```python
self.value_stream
```

輸出：

```text
V(s)
```

### (3) Advantage Stream

```python
self.advantage_stream
```

輸出：

```text
A(s,a)
```

---

## 5.3 Q-value 合成

```python
qvals = values + (
    advantages - advantages.mean(dim=1, keepdim=True)
)
```

---

## 5.4 Dueling DQN 優點

相比一般 DQN：

- 更容易學習 state importance
- 提高 policy 學習效率
- 對大量相似 action 更有效

# 7. Target Network

使用：

```python
target_model = copy.deepcopy(model)
```

並定期同步：

```python
if step_count % sync_freq == 0:
    target_model.load_state_dict(
        model.state_dict()
    )
```

目的：

- 提升 learning stability
- 避免 target 不斷改變
